Set up this project. Detects the full stack and conventions, asks two quick questions about your workflow preferences, and saves everything to `.skynex/project-config.yaml` so every agent session starts with context already loaded.

## Step 1 ‚Äî Detect project context

Read these files **if they exist** (never fail if missing):
- `CONVENTIONS.md`, `AGENTS.md` ‚Äî project conventions and context
- `package.json` ‚Äî runtime, framework, scripts (test/build/lint/typecheck)
- `go.mod`, `go.sum` ‚Äî Go module and dependencies
- `pyproject.toml`, `requirements.txt` ‚Äî Python stack
- `Cargo.toml` ‚Äî Rust stack
- `tsconfig.json` ‚Äî TypeScript compiler settings and path aliases
- `src/`, `cmd/`, `app/`, `internal/` top-level ‚Äî architecture pattern and module layout
- Test files in `src/` or `tests/` ‚Äî test framework and conventions
- `.eslintrc*`, `eslint.config.*`, `.prettierrc*`, `golangci.yml` ‚Äî linter config
- `.env.example` ‚Äî environment variables
- `Makefile` ‚Äî build/test/lint targets if no package.json scripts

From this, infer:
- `runtime`: node | go | python | rust | ruby | ...
- `language`: typescript | javascript | go | python | rust | ...
- `framework`: nestjs | express | fastapi | gin | react | vue | svelte | none | ...
- `test_runner`: vitest | jest | playwright | go-test | pytest | cargo-test | none
- `architecture`: ddd | mvc | modular | monolith | ...
- `commands.test`: exact command string (e.g. `pnpm test`, `go test ./...`, `pytest`)
- `commands.build`: exact command or omit if none found
- `commands.lint`: exact command or omit if none found
- `commands.typecheck`: exact command or omit if not applicable
